# 考公宝题库系统架构设计

> 适用：Taro 3.x + React + TS + 微信 CloudBase 云开发
> 题库：历年真题，共 5 大模块（常识判断、言语理解、数量关系、判断推理、资料分析）

---

## 1. 总体数据模型

```
questions          题目主表（所有模块统一存储，按 module_id 区分）
users              用户表
answer_records     答题记录表
wrong_book         错题本（可合并到 answer_records，独立表查询更快）
favorites          收藏表
progress           用户每日/总体学习进度表
```

---

## 2. 题目集合 `questions`（核心）

### 2.1 字段与类型

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `_id` | string | 云数据库自动 ID，也作为题目唯一标识 | `q_gk2000_language_001` |
| `module_id` | string | 模块标识 | `mod_language` |
| `year` | number | 年份 | `2000` |
| `source` | string | 来源 | `真题` / `模拟` |
| `type` | string | 题型：`single` 单选、`multiple` 多选、`subjective` 主观 | `single` |
| `difficulty` | string | 难度：简单/中等/困难 | `中等` |
| `content` | string | 题干（支持换行） | 真正的辩论与辩论比赛之不同... |
| `options` | array<string> | 选项文本 | `["A. xxx", "B. xxx"]` 或 `["xxx", "yyy"]` |
| `answer` | number / array<number> | 正确答案索引；多选为数组 | `0` 或 `[0, 2]` |
| `explanation` | string | 解析 | 这道题的主旨在于... |
| `commonErrors` | string | 常见错误点 | 易误选B，因为... |
| `tags` | array<string> | 标签 | `["言语理解", "主旨概括"]` |
| `points` | number | 分值，默认 1 | `1` |
| `createdAt` | Date | 导入时间 | 自动生成 |
| `updatedAt` | Date | 更新时间 | 自动生成 |

### 2.2 索引设计（CloudBase 控制台创建）

```json
[
  { "name": "module_year", "keys": { "module_id": 1, "year": 1 } },
  { "name": "year_module", "keys": { "year": 1, "module_id": 1 } },
  { "name": "module_id", "keys": { "module_id": 1 } },
  { "name": "year", "keys": { "year": -1 } },
  { "name": "source", "keys": { "source": 1 } },
  { "name": "difficulty", "keys": { "difficulty": 1 } }
]
```

> 说明：索引用于加速按模块、年份筛选；`module_id + year` 联合索引最常用。

## 4. 云函数设计

### 4.1 `questions` 云函数（题目主入口）

支持 action：

```js
{
  action: 'get_list',      // 获取题目列表
  action: 'get_detail',    // 获取题目详情
  action: 'get_for_practice', // 练习抽题
  action: 'get_explanation',  // 获取解析
  action: 'get_wrong',     // 获取错题（需配合 user）
  action: 'get_module_counts', // 模块数量
  action: 'search'         // 搜索题目
}
```

#### 4.1.1 `get_list` — 获取题目列表

```js
// 请求参数
{
  action: 'get_list',
  module_id: 'mod_language',  // 可选
  year: 2020,                 // 可选
  difficulty: '中等',          // 可选
  keyword: '关键词',           // 可选
  page: 1,                    // 页码，默认 1
  pageSize: 20                // 每页条数，默认 20
}

// 返回
{
  code: 0,
  data: {
    list: [...],
    total: 120,
    page: 1,
    pageSize: 20,
    has_more: true
  }
}
```

实现要点：

```js
const where = {};
if (module_id) where.module_id = module_id;
if (year > 0) where.year = year;
if (difficulty) where.difficulty = difficulty;
if (keyword) where.content = db.RegExp({ regexp: keyword, options: 'i' });

const total = (await db.collection('questions').where(where).count()).total;
const list = await db.collection('questions')
  .where(where)
  .field({ _id: true, module_id: true, year: true, type: true, difficulty: true, content: true, options: true, answer: true, explanation: true, tags: true, points: true })
  .skip((page - 1) * pageSize)
  .limit(pageSize)
  .orderBy('year', 'desc')
  .get();
```

#### 4.1.2 `get_for_practice` — 练习抽题

```js
// 请求参数
{
  action: 'get_for_practice',
  module_id: 'mod_language',  // 可选，不传则全部模块
  year: 2020,                 // 可选
  limit: 20                   // 抽取数量
}

// 返回
{
  code: 0,
  data: { list: [...], total: 5633, has_more: false }
}
```

实现要点：

```js
const total = (await db.collection('questions').where(where).count()).total;
const fetchLimit = Math.min(limit, total);
const randomSkip = total > fetchLimit ? Math.floor(Math.random() * (total - fetchLimit)) : 0;

const res = await db.collection('questions')
  .where(where)
  .field({ ... })
  .skip(randomSkip)
  .limit(fetchLimit)
  .get();
```

> 注意：随机抽取用 `skip` 近似，大数据量时建议用 `aggregate().sample()`。

#### 4.1.3 `get_explanation` — 获取解析

```js
{
  action: 'get_explanation',
  question_id: 'q_gk2000_language_001'
}
```

返回 `explanation`、`commonErrors`、`answer`、`analysis`。

### 4.2 `submit` 云函数（提交答案）

```js
{
  action: 'submit',
  question_id: 'q_gk2000_language_001',
  userAnswer: 0,              // 或 [0, 2] 或 "主观文字"
  duration: 15,               // 答题耗时（秒）
```

云函数内部逻辑：

1. 查询题目正确答案
2. 判题（见第 5 节）
3. 写入 `answer_records`
4. 若答错，写入/更新 `wrong_book`
5. 更新 `progress` 表
6. 返回结果

```js
{
  code: 0,
  data: {
    isCorrect: true,
    correctAnswer: 0,
    score: 1,
    explanation: '...',
    commonErrors: '...'
  }
}
```

## 5. 判题与评分逻辑

### 5.1 单选题

```js
function judgeSingle(correct, user) {
  return correct === user;
}
```

得分：答对得 `points` 分，答错 0 分。

### 5.2 多选题

```js
function judgeMultiple(correct, user) {
  if (!Array.isArray(correct) || !Array.isArray(user)) return false;
  if (user.length === 0) return false;
  if (correct.length === user.length) {
    return correct.every(c => user.includes(c));
  }
  return false;
}
```

> 严格模式：全对才给分；也可按部分给分（如选对 1 个给 0.5 分）。

### 5.3 主观题（概括归纳）

主观题建议先在前端展示参考答案，用户自行核对评分；或调用 AI 能力评分。

```js
function judgeSubjective(correctText, userText) {
  if (!userText || !correctText) return { isCorrect: false, score: 0 };
  
  // 简单关键词匹配
  const correctKeywords = extractKeywords(correctText);
  const matched = correctKeywords.filter(k => userText.includes(k)).length;
  const ratio = matched / correctKeywords.length;
  
  const score = Math.round(ratio * points * 10) / 10; // 保留一位小数
  return {
    isCorrect: score >= points * 0.8,  // 80% 关键词覆盖算对
    score,
    matchedKeywords: matched
  };
}

function extractKeywords(text) {
  // 示例：提取关键短语，实际可用分词服务或 AI 接口
  return [...new Set(text.match(/[\u4e00-\u9fa5]{2,8}/g) || [])];
}
```

> 高阶方案：接入 CloudBase AI / 腾讯文心等，对主观题做语义相似度评分。

## 6. 分页与筛选逻辑

### 6.1 分页

```js
const page = Math.max(1, event.page || 1);
const pageSize = Math.min(50, Math.max(1, event.pageSize || 20));
const skip = (page - 1) * pageSize;

const res = await db.collection('questions')
  .where(where)
  .skip(skip)
  .limit(pageSize)
  .orderBy('year', 'desc')
  .get();

const total = (await db.collection('questions').where(where).count()).total;
const has_more = skip + res.data.length < total;
```

### 6.2 筛选条件组合

```js
const where = {};
if (module_id) where.module_id = module_id;
if (year > 0) where.year = year;
if (difficulty) where.difficulty = difficulty;
if (source) where.source = source;
if (keyword) {
  where.content = db.RegExp({
    regexp: keyword.replace(/[.*+?^${}()|[\]\\]/g, '\\$&'),
    options: 'i'
  });
}
```

### 6.3 年份筛选

- 全部年份：不传 `year` 或传 `year: 0`
- 某一年：`year: 2020`
- 近 5 年：`where.year = _.gte(2021)`

## 7. 权限控制

### 7.1 用户数据权限

CloudBase 数据库安全规则示例：

```json
{
  "read": "doc._openid == auth.openid",
  "write": "doc._openid == auth.openid"
}
```

应用于集合：`users`, `answer_records`, `wrong_book`, `favorites`, `progress`。

`questions` 为公开题库，不设权限限制。

### 7.2 云函数中校验

```js
const cloud = require('wx-server-sdk');
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV });

exports.main = async (event, context) => {
  const { OPENID } = cloud.getWXContext();
  const openid = event._openid || OPENID;
  
  if (!openid) {
    return { code: 401, message: '未获取到用户身份' };
  }
  
  // 所有涉及用户数据的查询都加 _openid: openid
};
```

### 7.3 关键权限点

- 用户只能查看/修改自己的答题记录
- 用户只能查看/修改自己的错题本和收藏
- 用户只能查看自己的进度
- 管理员（可选）通过云函数中配置 `adminOpenids` 白名单，可查询全部数据

```js
const ADMIN_OPENIDS = ['oXXXXX1', 'oXXXXX2']; // 管理员 openid
const isAdmin = ADMIN_OPENIDS.includes(openid);
```

```ts
  tags: string[];
  points: number;
  userAnswer?: number | number[] | string;

  duration: number;

}
```
