# 考公宝 V2.0 云函数重构设计

> 版本：v2.0-draft-1  
> 范围：管理后台、PDF/MinerU 导入任务、草稿逐题审核、Gemini 审核、V2 组包发布、用户端题库查询  
> 关联文档：[考公宝V2.0题库导入系统重构方案.md](./考公宝V2.0题库导入系统重构方案.md)

## 1. 结论

当前项目不需要拆成几十个独立云函数，也不需要推翻 `admin` 云函数。第一阶段采用：

```text
admin 云函数（统一管理入口）
  ├─ features/import-tasks.js   导入任务
  ├─ features/draft-papers.js   草稿试卷
  ├─ features/question-drafts.js 一题一档
  ├─ features/ai-review.js      Gemini 异步审核
  ├─ features/release.js        校验、组包、发布
  └─ features/xingce.js         V2 包最终校验和正式入库（保留）

workerGateway 云函数（本机 MinerU 专用入口）
  ├─ claim
  ├─ heartbeat
  ├─ upload_result
  ├─ complete
  └─ fail

questions 云函数（小程序只读题库接口，保留）
```

这样能保留现有 CloudBase 架构，同时把管理员、Worker 和普通用户三种调用权限隔离。

## 2. 现状核对

### 2.1 已完成的能力

- `cloudfunctions/importQuestions/index.js` 已经把旧的 `importQuestions1~12` 合并成一个通用兼容函数。
- `cloudfunctions/admin/features/xingce.js` 已具备 V2 包校验和正式入库能力，继续作为最终发布器。
- `cloudfunctions/admin/features/drafts.js` 已实现创建、追加、查看、修改、通过、驳回、Gemini 审核和发布。
- `cloudfunctions/questions/index.js` 已作为小程序端的只读查询入口。

### 2.2 当前主要问题

| 问题 | 当前表现 | 重构目标 |
|---|---|---|
| 草稿存储过大 | 一份试卷的完整 `package`、审核状态和修改内容塞在一个文档 | `draft_papers` 存试卷；`question_drafts` 一题一档 |
| 请求体超限 | 整套 V2 包一次写入，曾出现 `EXCEED_MAX_PAYLOAD_SIZE` | 小批量写题；图片、Markdown、layout JSON 放云存储 |
| AI 审核同步 | 浏览器等待 Gemini 返回 | 创建异步任务，后台轮询进度 |
| 发布耦合 | 草稿模块直接组包并写正式库 | 独立 `release` 服务，锁定、校验、组包、发布、记录结果 |
| 权限粗糙 | `ADMIN_OPENIDS` 或一个 `ADMIN_SECRET` 拥有全部权限 | RBAC + Worker 独立签名密钥 |
| 错误不可追踪 | 返回 `code/message`，无稳定业务码和请求号 | 统一 `request_id`、`error_code`、字段级错误 |
| 用户端默认答案 | 查询格式化逻辑可能把缺失答案回退为 `0` | 未发布题不得进入正式库；正式库答案必须已验证 |

> 注意：仓库里现在只有一个 `importQuestions` 云函数。`batch_01~12` 是历史批次数据，不是 12 个正在运行的云函数。

## 3. 云函数边界

### 3.1 `admin`：管理端网关

职责：

- 管理员身份与角色校验；
- 参数校验、动作路由；
- 统一响应、错误码、审计上下文；
- 调用领域 feature，不直接堆业务代码；
- 处理短请求，长任务只创建任务并立即返回。

不负责：

- 运行 MinerU；
- 在一次请求里处理整份 PDF；
- 接收大图片二进制或完整 layout JSON；
- 让 AI 直接写正式题库。

### 3.2 `workerGateway`：本机 Worker 网关

职责：

- 本机 MinerU Worker 领取任务；
- 上报心跳、页数和阶段进度；
- 登记上传到云存储后的文件 ID；
- 小批量提交一题一档数据；
- 完成或标记失败。

该云函数不能使用 `ADMIN_SECRET`，必须使用独立 `WORKER_SECRET` 或 HMAC 签名。

### 3.3 `questions`：用户端只读服务

职责：

- 查询已发布、`status=enabled` 的题目；
- 查询试卷、模块、练习题；
- 输出兼容当前小程序的扁平字段，同时逐步增加 V2 blocks。

禁止：

- 读取 `question_drafts`；
- 返回未验证答案；
- 接受管理端写操作。

### 3.4 `importQuestions`：仅保留为旧数据兼容器

该函数仅处理历史批次迁移和修复，不再作为 OCR 新题录入入口。`import_inline` 后续应返回弃用提示，OCR 结果必须进入 `import_tasks → question_drafts`。

## 4. 统一请求与响应

### 4.1 管理端请求

```json
{
  "action": "question_draft.update",
  "request_id": "req_1784_xxx",
  "idempotency_key": "edit_qd_xxx_v3",
  "data": {
    "question_id": "qd_paper001_066",
    "version": 3,
    "patch": {}
  }
}
```

兼容期内允许旧的扁平参数，但新页面只发送 `data`。

### 4.2 成功响应

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "request_id": "req_1784_xxx"
}
```

### 4.3 失败响应

```json
{
  "code": 422,
  "error_code": "DRAFT_VALIDATION_FAILED",
  "message": "题目仍有待修正项",
  "details": [
    { "path": "answer.value", "message": "答案缺失" }
  ],
  "request_id": "req_1784_xxx"
}
```

### 4.4 业务错误码

| HTTP/业务 code | error_code | 含义 |
|---:|---|---|
| 400 | `INVALID_REQUEST` | 参数格式错误 |
| 401 | `UNAUTHORIZED` | 缺少有效身份 |
| 403 | `FORBIDDEN` | 角色无权执行 |
| 404 | `RESOURCE_NOT_FOUND` | 任务、草稿或题目不存在 |
| 409 | `VERSION_CONFLICT` | 乐观锁版本冲突 |
| 409 | `TASK_ALREADY_CLAIMED` | Worker 任务已被领取 |
| 413 | `PAYLOAD_TOO_LARGE` | 请求体超过安全限制 |
| 422 | `DRAFT_VALIDATION_FAILED` | 草稿字段或 V2 包校验失败 |
| 422 | `ANSWER_NOT_VERIFIED` | 答案未确认 |
| 429 | `RATE_LIMITED` | AI/接口调用过快 |
| 500 | `INTERNAL_ERROR` | 未预期异常 |
| 503 | `AI_PROVIDER_UNAVAILABLE` | Gemini 暂时不可用 |

## 5. 权限设计

### 5.1 角色

- `super_admin`：系统配置、删除、回滚、全部权限；
- `editor`：创建任务、编辑草稿、上传资源；
- `reviewer`：逐题通过、驳回、调用 AI 审核；
- `publisher`：整卷预检和发布；
- `worker`：领取和提交 MinerU 任务；
- `user`：仅使用小程序正式题库。

### 5.2 权限矩阵

| 动作 | editor | reviewer | publisher | super_admin | worker |
|---|:---:|:---:|:---:|:---:|:---:|
| 创建/取消导入任务 | ✓ |  |  | ✓ |  |
| 编辑题目 | ✓ | ✓ |  | ✓ |  |
| AI 审核 | ✓ | ✓ |  | ✓ |  |
| 人工通过/驳回 |  | ✓ |  | ✓ |  |
| 发布/回滚 |  |  | ✓ | ✓ |  |
| 领取/上报 MinerU 任务 |  |  |  |  | ✓ |

管理员身份建议存入 `admin_users`：

```json
{
  "_id": "openid_xxx",
  "roles": ["editor", "reviewer"],
  "status": "enabled",
  "updated_at": "Date"
}
```

环境变量 `ADMIN_OPENIDS` 只作为首个超级管理员的应急入口，不作为长期角色系统。

### 5.3 Worker 请求签名

请求头：

```text
X-Worker-Id: pc-5070-01
X-Timestamp: 1784100000
X-Nonce: random
X-Signature: HMAC_SHA256(WORKER_SECRET, timestamp + "\n" + nonce + "\n" + sha256(body))
```

服务端拒绝超过 5 分钟的时间戳，并将 nonce 短期写入 `worker_nonces` 防重放。

## 6. Action 目录

### 6.1 导入任务

| 新 action | 权限 | 说明 |
|---|---|---|
| `import_task.create` | editor | 创建题目 PDF + 答案 PDF 配对任务 |
| `import_task.list` | editor/reviewer | 分页查询任务 |
| `import_task.get` | editor/reviewer | 查询阶段、进度、错误 |
| `import_task.cancel` | editor | 取消未发布任务 |
| `import_task.retry` | editor | 从失败阶段重试 |

创建任务只登记云存储文件，不上传 PDF 内容到云函数：

```json
{
  "action": "import_task.create",
  "data": {
    "paper_name": "2025国考行政执法卷",
    "question_pdf_file_id": "cloud://.../question.pdf",
    "answer_pdf_file_id": "cloud://.../answer.pdf",
    "question_pdf_sha256": "...",
    "answer_pdf_sha256": "..."
  }
}
```

### 6.2 草稿试卷

| 新 action | 说明 |
|---|---|
| `draft_paper.list` | 草稿箱整卷列表 |
| `draft_paper.get` | 试卷统计和资料组 |
| `draft_paper.validate` | 检查题号、题量、资料组、资源和答案解析匹配率 |
| `draft_paper.archive` | 归档，不直接物理删除 |

### 6.3 一题一档

| 新 action | 说明 |
|---|---|
| `question_draft.list` | 按试卷、状态、模块、问题类型分页 |
| `question_draft.get` | 获取结构化题、证据和审核历史 |
| `question_draft.update` | 乐观锁修改题干、选项、答案、解析、图片 |
| `question_draft.approve` | 人工通过一题 |
| `question_draft.reject` | 驳回并要求填写原因 |
| `question_draft.bulk_approve` | 仅通过已无错误且答案已验证的题 |
| `question_draft.recheck` | 重新运行确定性校验 |

更新时必须传 `version`。成功后 `version + 1`；版本不一致返回 `VERSION_CONFLICT`，防止两个审核员互相覆盖。

### 6.4 AI 审核

| 新 action | 说明 |
|---|---|
| `ai_review.create` | 创建单题或小批量 Gemini 审核任务 |
| `ai_review.get` | 查询任务进度与建议 |
| `ai_review.apply` | 人工选择性应用 AI 建议 |
| `ai_review.retry` | 重试失败任务 |

AI 只能写 `ai_review_results`，不能自动修改题目，也不能把题目标为人工通过。

### 6.5 发布

| 新 action | 说明 |
|---|---|
| `release.preview` | 组装 V2 包并调用 `previewXingcePackage` |
| `release.create` | 创建发布任务并立即返回 `release_id` |
| `release.get` | 查询发布进度和结果 |
| `release.rollback` | 超级管理员按发布版本回滚 |

### 6.6 Worker actions

| action | 说明 |
|---|---|
| `claim` | 原子领取一个 waiting/retry 任务 |
| `heartbeat` | 续租并上报阶段进度 |
| `register_assets` | 登记 Markdown、layout JSON、页图、题图 file_id |
| `upsert_questions` | 每批最多 10~20 道题写入草稿 |
| `complete` | 完成 MinerU/切题阶段 |
| `fail` | 写入可定位的失败阶段和错误摘要 |

## 7. 状态机与并发控制

### 7.1 导入任务状态

```text
waiting
  → claimed
  → mineru_processing
  → splitting
  → draft_ready
  → ai_reviewing（可选）
  → human_review
  → ready_to_publish
  → publishing
  → published
```

任一处理阶段可以进入 `failed`；未发布前可以进入 `cancelled`。

### 7.2 Worker 租约

`import_tasks` 增加：

```json
{
  "worker_id": "pc-5070-01",
  "lease_token": "lease_xxx",
  "lease_expires_at": "Date",
  "heartbeat_at": "Date",
  "retry_count": 0
}
```

`claim` 必须通过事务或条件更新保证只有一个 Worker 成功。心跳超时后，调度器才允许重新领取。

### 7.3 幂等

- 创建导入任务：以两个 PDF SHA-256 + 试卷类型生成 `dedupe_key`；
- 草稿题：固定 `_id = qd_<draftPaperId>_<questionNo>`；
- 发布：固定 `release_key = <draftPaperId>:<version>`；
- 正式入库：继续利用 `xingce.js` 的稳定 paper/question ID upsert；
- 同一 `idempotency_key` 重试时返回第一次结果，不重复写库。

## 8. 大文件与 CloudBase 限制

### 8.1 绝不放进云数据库文档的内容

- PDF 二进制；
- 页图、题图、解析图；
- 完整 MinerU Markdown；
- 完整 layout/content_list JSON；
- 整套 V2 package。

这些内容放云存储，数据库只保存 `file_id`、SHA-256、页码、bbox 和元数据。

### 8.2 写入批次

- Worker 每批提交 10 道题，最大不超过 20 道；
- 单次请求先计算 JSON 字节数，超过安全阈值自动缩小批次；
- 图片先直传云存储，再提交 file_id；
- 不允许用 Base64 把图片塞进请求；
- 发布组包在服务端按分页读取草稿，不从浏览器上传整套包。

## 9. 发布事务

发布步骤：

```text
1. release.create 获取发布锁
2. 分页读取 draft_papers / question_drafts / draft_assets
3. 校验所有题 review_status=approved
4. 校验 answer.value 为 0-3 且 answer.verified=true
5. 校验解析、图片资源、资料分析 1+5 关系
6. 组装临时 V2 package
7. 调用 previewXingcePackage
8. 预检通过后调用 importXingcePackage
9. 写 release_jobs 和 review_events
10. 标记 draft_paper=published
```

发布失败时保留草稿和发布日志，不把草稿标为 published。由于 CloudBase 无法对跨集合大批量写入做一个超大事务，正式集合采用稳定 ID 的幂等 upsert；重试同一 `release_key` 不产生重复题。

## 10. 当前 action 到新 action 的迁移

| 当前调用 | 新调用 | 兼容策略 |
|---|---|---|
| `action=draft, draft_action=create` | `draft_paper.create_from_v2` | 仅迁移期保留整包入口，服务端立即拆成一题一档 |
| `draft/append` | `question_draft.batch_upsert` | 每批 10~20 题 |
| `draft/list` | `draft_paper.list` | 前端改一次 |
| `draft/get` | `draft_paper.get` + `question_draft.list` | 分页加载，避免大响应 |
| `draft/update` | `question_draft.update` | 增加 version 乐观锁 |
| `draft/approve` | `question_draft.approve` | 逐题状态落题目文档 |
| `draft/reject` | `question_draft.reject` | 驳回原因写 review_events |
| `draft/gemini_review` | `ai_review.create` | 改为异步任务 |
| `draft/publish` | `release.create` | 改为发布任务 |
| `preview_xingce_package` | 内部 `release.preview` | 管理页面不再上传完整包 |
| `import_xingce_package` | 内部发布器 | 仅 release 服务调用 |
| `importQuestions/import_inline` | 禁用新 OCR 录入 | 返回弃用提示并引导草稿链路 |

兼容入口保留两个发布版本；后台全部迁移后再删除旧 action。

### P3：发布服务

- 为旧批次保留只读迁移工具。

## 14. 验收标准

- 题目 PDF 和答案解析 PDF 能创建一条配对任务；
- 本机 MinerU 中断后可继续，不重复生成题目；
- 130 题试卷不会再触发请求体或单文档大小限制；
- 草稿箱可按错误类型筛选并逐题修改；
- 两个审核员同时编辑时能检测版本冲突；
- Gemini 失败不影响人工审核，且不会自动改题或通过；
- 无答案、答案未验证、图片未上传、资料组不是 5 题时无法发布；
- 发布重试不会生成重复试卷或重复题；
- 小程序只能读取已发布正式题库；
- 每次修改、审核、发布都有操作日志和 `request_id` 可追踪。

## 15. 第一批代码改造顺序

第一批只做基础设施，不立刻删除现有功能：

1. 新增 `admin/lib/errors.js`、`response.js`、`auth.js`；
2. 在 `admin/index.js` 接入请求上下文和 RBAC，同时兼容旧响应；
3. 新增 `features/import-tasks.js`；
