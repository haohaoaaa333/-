# 考公宝 V2.0 数据库索引与权限清单

> 配套《考公宝 V2.0 云函数重构设计》。本文件是「索引 + ACL 权限」的单一事实源与落地清单。
> `cloudbaserc.json` 已声明 `database.collections` 的 acl；索引建议在控制台 / CLI 手动创建（见下文）。

---

## 一、必须建立的索引

### 1. `questions`（正式题库，高频读）
| 字段 | 方向 | 说明 |
|---|---|---|
| `paper_id` | 升序 | 按卷筛选 |
| `type` | 升序 | 按题型筛选 |
| `knowledge_points` | 升序（多值/数组） | 知识点薄弱度分析 |
| `difficulty` | 升序 | 难度筛选 |

### 2. `question_drafts`（草稿，审核页高频）
| 字段 | 方向 | 说明 |
|---|---|---|
| `paper_id` | 升序 | 按草稿卷列题 |
| `review_status` | 升序 | pending/approved/rejected 过滤 |
| `question_no` | 升序 | 题号排序 |

### 3. `practice_records`（单题答题，学习分析）
| 字段 | 方向 | 说明 |
|---|---|---|
| `user_id` / `_openid` | 升序 | 我的答题记录 |
| `question_id` | 升序 | 某题全用户体验 / 已答标记 |

### 4. 补充索引（性能与聚合）
| 集合 | 字段 | 说明 |
|---|---|---|
| `wrong_questions` | `user_id` + `question_id` | 错题本查询 + 幂等 upsert |
| `favorites` | `user_id` + `question_id` | 收藏列表 + 幂等切换 |
| `import_tasks` | `status` | Worker 拉取待处理任务 |
| `knowledge_points` | `paper_ids` | 知识点聚合反查 |
| `draft_assets` | `question_id` | 证据链回查 |

> 多值字段（`knowledge_points` / `paper_ids`）建索引时勾选「数组 / 多值」。

---

## 二、建索引方法

### 方式 A：云开发控制台（推荐，最直观）
1. 云开发控制台 → 数据库 → 选择集合
2. 索引管理 → 新建索引
3. 填写字段名、排序（1=升序）、是否唯一、是否数组（多值）

### 方式 B：CLI（tcb / cloudbase）
```bash
# 单字段
tcb db index create questions paper_id 1
tcb db index create questions type 1
tcb db index create question_drafts review_status 1
tcb db index create practice_records _openid 1
tcb db index create wrong_questions user_id 1
tcb db index create favorites user_id 1
# 多值字段（数组）
tcb db index create questions knowledge_points 1
```
> 具体子命令以当前 CloudBase CLI 版本为准；不确定时优先用控制台。

## 三、集合权限（ACL）

`aclTag` 含义（CloudBase 数据库权限标签）：
- **READONLY**：所有用户可读，仅管理端（服务端 SDK）可写 → 适合公开题库
- **PRIVATE**：仅创建者可读写 → 适合用户私有数据
- **ADMINWRITE**：所有用户可读，仅管理端可写（等价 READONLY 的写入侧）
- **WRITEONLY**：仅创建者可写

> 说明：ACL 仅约束**小程序端直连数据库**。所有云函数（`admin`/`practice`/`user`/`questions` 等）通过服务端 SDK 访问，**不受 ACL 限制**，因此 `PRIVATE` 的草稿/审核集合仍可被 `admin` 正常读写。

| 集合 | aclTag | 理由 |
|---|---|---|
| `questions` | READONLY | 公开题库，用户刷题只读 |
| `papers` | READONLY | 公开卷信息 |
| `knowledge_points` | READONLY | 学习分析需公开读 |
| `categories` / `banners` | READONLY | 配置类公开 |
| `vip_plans` / `book_packs` | READONLY | 商品/题包公开 |
| `essay_papers` / `essay_materials` / `essay_questions` / `essay_answers` | READONLY | 申论公开内容 |
| `practice_records` | PRIVATE | 我的单题答题 |
| `practice_sessions` | PRIVATE | 我的练习会话 |
| `wrong_questions` | PRIVATE | 我的错题本 |
| `favorites` | PRIVATE | 我的收藏 |
| `user_progress` | PRIVATE | 我的进度 |
| `users` | PRIVATE | 我的账户（经 user 云函数访问） |
| `draft_papers` / `question_drafts` / `draft_assets` / `review_events` | PRIVATE | 草稿/审核，仅 admin 服务端访问 |
| `import_tasks` / `import_jobs` | PRIVATE | 导入任务，仅 admin / Worker 服务端 |
| `admin_users` | PRIVATE | 管理员账户，仅 admin 服务端 |
| `orders` / `book_upload_chunks` / `essay_import_jobs` | PRIVATE | 订单/上传/导入，私密 |

> 上述 acl 已在 `cloudbaserc.json` 的 `database.collections` 声明，部署时自动应用；若当前 CLI 版本不自动应用，可在控制台手动设置（集合 → 权限设置 → 选择对应标签）。
