## 2026-07-09 改用 CloudBase v3 官方网页版 ESM（jsdelivr +esm）

问题：自检显示 `appAuthType: 'function'`、`appStorageType: 'object'`，上传报 `app.storage is not a function`。
根因：`static.cloudbase.net` 上的 `cloudbase-js-sdk/3.6.2/*.js` 虽是 v3，但它是**小程序/Node 构建**（browser 分支下 `app.storage` 被实现为只有 `http` 的伪对象），不是真正的网页版 SDK。网页版 SDK 是 npm 包 `@cloudbase/js-sdk`，v3 已改为 ESM 模块化。

- 存储扩展：`https://cdn.jsdelivr.net/npm/@cloudbase/js-sdk@3.6.2/storage/dist/index.esm.js/+esm`（export registerStorage）
- v3 网页版 `uploadFile` 参数是 `filePath`（File 对象），不是 v1 的 `fileContent`。
改动：
- 删除本地 `admin/cloudbase-v3-cloudbase.js` 等三个小程序/Node 构建文件。
- `admin/index.html`：改为 `<script type="module">` 从 jsdelivr `+esm` 引入核心+auth+storage，显式 `registerAuth(app)`、`registerStorage(app)`，并把 `window.cloudbase` 与 `window.__cloudbaseApp` 暴露给 `app.js`；自检脚本额外探测 `storageInstance.uploadFile` 是否存在。
- `admin/app.js`：`getCloudbaseApp()` 优先复用 `window.__cloudbaseApp`；`uploadBookFileToStorage` 兼容 `app.storage` 是函数或对象的情况，统一用 `storage.uploadFile({ cloudPath, filePath: file })`。
- `app.js` 改为 `<script type="module" src="./app.js">` 加载，确保在 CloudBase SDK module 初始化之后执行。
