### importQuestions 云函数拆分（解决上传大小限制）

- 用户尝试上传 `importQuestions` 遇到 "400 input length too long" — 原因为云函数含 12 个 batch JSON 文件，总计 5.8MB，超过 CloudBase 上传大小限制。
- **方案**：拆分为 1 个编排器 + 4 个子函数，每个子函数含 2-4 个批次，全部控制在 2MB 以内。
- 新建 4 个子云函数：

| 云函数 | 实际批次 | 数据量 |
|--------|---------|--------|
| importQuestions1 | batch 1-3 | 1013KB (~1.0MB) |
| importQuestions2 | batch 4-6 | 1.7MB |
| importQuestions3 | batch 7-8 | 1.5MB |
| importQuestions4 | batch 9-12 | 1.6MB |

- 每个子函数的 `index.js` 含 `BATCH_MAP` 配置，映射内部编号到实际批次号：
  - 调用 `{ action: "import", batch: 1 }` → 子函数内部映射到真实 batch_XX.json
  - 调用 `{ action: "import_all" }` → 该子函数负责的全部批次
- 原始 `importQuestions` 重写为轻量级编排器（9KB）：
  - `{ action: "import", batch: N }` → 查询 `BATCH_ROUTER` 映射表，路由到对应子函数
  - `{ action: "import_all" }` → 串行调用 4 个子函数的 `import_all`
  - `{ action: "stats" }` → 直接查询 DB，额外输出 `by_year` 分组统计
- 从原始目录移除了 batch_*.json 文件，清理出 ~5.8MB 空间。
- **待用户操作**：在微信开发者工具右键每 5 个云函数目录 → 「上传并部署」
