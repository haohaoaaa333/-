## 修复：历年真题跳转问题

### 问题
点击首页"历年真题"按钮，跳转无变化，页面不显示。

### 根因
1. **`pages/history/index` 未注册**：新创建的模块分类页面没有在 `app.config.ts` 的 `pages` 数组中注册，导致 `navigateTo` 找不到页面
2. **首页跳转 URL 错误**：`home/index.tsx` 中 `type === 'history'` 时跳转的是 `/pages/papers/index` 而非 `/pages/history/index`

### 修复
- `src/app.config.ts`：在 pages 数组中添加 `'pages/history/index'`
- `src/pages/home/index.tsx`：将 历年真题 跳转从 `/pages/papers/index` 改为 `/pages/history/index`
- 编译通过（7.81s）

### 待办

- 部署后清除缓存：工具 → 清除缓存 → 清除全部
- 重新编译即可正常使用历年真题功能

## 修复：400 错误 — 云函数过大无法上传

### 问题
importQuestions4 (1.9MB) 和 importQuestions6 (1.9MB) 超过 CloudBase 上传限制，部署时报 400 "input length too long"。

### 根因
之前每个 importQuestions 含 2 个 batch JSON 文件，合起来太大。

### 修复方案（v3）
拆分为 12 个子云函数，每个只含 1 个 batch 文件：

| 函数 | Batch | 大小 |
|------|-------|------|
| importQuestions1  | batch_01 | 569KB |
| importQuestions2  | batch_02 | 597KB |
| importQuestions3  | batch_03 | 713KB |
| importQuestions4  | batch_04 | 817KB |
| importQuestions5  | batch_05 | 805KB |
| importQuestions6  | batch_06 | 857KB |
| importQuestions7  | batch_07 | 977KB |
| importQuestions8  | batch_08 | 973KB |
| importQuestions9  | batch_09 | 761KB |
| importQuestions10 | batch_10 | 925KB |
| importQuestions11 | batch_11 | 1.2MB |
| importQuestions12 | batch_12 | 737KB |

全部 < 1.2MB，远低于上传限制。

### 修改文件
- `cloudfunctions/importQuestions1-6/index.js`：改为只 require 一个 batch
- `cloudfunctions/importQuestions1-6/`：移除多余 batch 文件
- `cloudfunctions/importQuestions7-12/`：新建（index.js + package.json + batch）
- `cloudfunctions/importQuestions/index.js`：编排器更新为 12 步计划

### 部署步骤
1. 在微信开发者工具中，逐个右键 importQuestions 云函数 → "上传并部署：云端安装依赖"
2. 先部署 1-6（已存在，需更新），再部署 7-12（新建）
3. 都部署完成后：调用 importQuestions → `{ "action": "clear", "confirm": true }`
4. 逐个调用 importQuestions1-12（无参数或 `{}`），每调用完一个检查返回值
5. 最后调用 importQuestions → `{ "action": "stats" }` 验证 ~5,820 题
